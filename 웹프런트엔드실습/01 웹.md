# 1주차

### 웹서비스 프로세스

웹을 통해서 클라이언트의 요청(Request)를 받아 서버에서 처리하고 응단(Response)하는 것.

🔸 클라이언트(client) : 웹을 통해서 서비스를 요청하는 사용자(컴퓨터)

클라이언트쪽에서 실행되는 기술 : 프런트엔드

🔸 서버(server) : 웹을 통해서 서비스를 제공하는 컴퓨터

서버쪽에서 실행되는 기술 : 백엔드

🔸 웹브라우저 : 서버에서 정보를 가져와 HTML, 이미지, 동영상 등 콘텐츠를 사용자의 화면에 보여주는 응용 프로그램(크롬, 사파리, 엣지 등)

### 웹페이지 구성 3요소

🔸 HTML(Hyper Text Markup Language)

웹 페이지의 구조와 내용

🔸 CSS(Cascading Style Language)

웹 페이지의 모양

🔸 Javascript(JS)

웹 페이지의 행동

### 정적 웹페이지 VS 동적 웹페이지

🔸 정적(static)인 컨텐츠 웹페이지

변하지 않는 컨텐츠

🔸 동적(dynamic)인 컨텐츠 웹페이지

동적으로 만들어진 변화있는 컨텐츠

---

### 단어 뜻

🔸 IP주소

인터넷(http)에 연결되어 있는 디바이스는 각 ip주소를 갖는다.

🔸 돔트리

웹 페이지의 HTML 구조를 부모와 자식 관계를 가진 **계층적인 나무(Tree) 구조**로 형상화한 것

트리를 구성하는 각 객체를 노드라고 한다.

🔸 API

서로 다른 소프트웨어 구성 요소가 상호작용할 수 있게 해주는 인터페이스이다.

내부 구현은 몰라도 규칙에 맞춰 요청하면 원하는 결과를 얻을 수 있다.

🔸 라이브러리 모듈

특정 기능을 수행하는 코드의 집합체. 개발자가 주도권을 가지고 **"내가 필요할 때"** 가져다 사용한다.

모듈 : 아주 작은 단위의 부품

라이브러리 : 모듈을 모아놓은 도구 상자

🔸 프레임워크

애플리케이션 개발을 위한 전체적인 구조를 제공.

개발자가 프레임워크에 맞춰 코드를 작성하면, **"프레임워크가 주도권을 가지고"** 개발자의 코드를 실행.

### MPA vs SPA

웹 애플리케이션이 화면을 사용자에게 보여주는 방식(아키텍처)에 따른 두 가지 대표적인 모델.

사용자가 페이지를 이동할 때 브라우저가 어떻게 동작하는지의 차이

🔸 mpa(multi page application)

여러 개의 페이지로 구성된 전통적인 방식

페이지를 이동할 때마다 서버에 새로운 HTML 요청

페이지를 이동할 때마다 새로고침되는 현상 발생

🔸 spa(single page application)

단 하나의 페이지로 구성된 현대적인 방식

페이지 이동 시 자바스크립트를 이용해 필요한 데이터만 받아와서 화면의 특정 부분만 갈아 끼움

페이지 이동 시 새로고침 없이 매끄러운 사용자 경험을 제공

| **구분** | **MPA (전통적 방식)** | **SPA (현대적 방식)** |
| --- | --- | --- |
| **렌더링 방식** | SSR (서버 사이드 렌더링) 중심 | CSR (클라이언트 사이드 렌더링) 중심 |
| **새로고침** | 이동할 때마다 발생 (깜빡임) | 발생하지 않음 (부드러움) |
| **검색 엔진 최적화(SEO)** | 매우 유리함 (모든 정보가 HTML에 있음) | 상대적으로 불리함 (추가 설정 필요) |
| **초기 로딩 속도** | 빠름 (당장 필요한 것만 받음) | 느림 (전체 코드를 처음에 다 받음) |
| **대표 기술** | JSP, PHP, ASP.NET | **React**, Vue, Angular |
