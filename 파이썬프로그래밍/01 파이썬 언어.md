# 1주차 파이썬 언어

### 파이썬 언어 등장

Guido van Rossum

1991년 발표

뱀이 아니라 Monty Python 코미디 그룹에서 따온 이름

### 파이썬 장점

- 배우기 쉬운 언어
- 읽기 쉬운 코드
- 빠른 개발 속도

### 파이썬이 많이 사용되는 분야

인공지능 - 머신러닝, 딥러닝

데이터 분석 - pandas, numpy

웹 개발 - Django, Flask

자동화 - 업무 자동화

게임 개발 - pygame

→ 왜 AI분야에서 Python을 많이 사용하는가?

라이브러리가 많고 개발속도가 빠르기 때문에

### C언어와 Python

|  | C언어 | Python |
| --- | --- | --- |
| 실행 방식 | 컴파일  | 인터프리터 |
| 자료형 | 정적 타입 | 동적 타입 |
| 문법 | 복잡 | 간결 |
| 코드 길이 | 길다 | 짧다 |

### 파이썬 코드의 특징

- 들여쓰기 기반 구조
    
    { } 사용하지 않고 들여쓰기 문법이다.
    
- 동적 타입
    
    변수 타입을 선언하지 않는다. 실행 시 타입이 결정된다.
    

### 인터프리터 VS 컴파일

- 컴파일 방식
    
    소스코드 → 컴파일 → 실행 파일(기계어) → 실행
    
    소스코드를 기계어로 한 번에 변환한 후 실행하는 방식
    
    - 장점
        - 실행 속도가 빠르다.
        - 실행 전에 오류를 발견할 수 있다.
        - 대규모 시스템 개발에 적합하다
    - 단점
        - 수정 후 다시 컴파일해야 한다.
        - 개발 과정이 느리다.

- 인터프리터 방식
    
    소스 코드를 한 줄씩 읽으면서 바로 실행하는 방식
    
    - 장점
        - 컴파일 과정이 없다.
        - 코드 작성 후 바로 실행가능하다.
        - 개발 속도가 빠르다.
        - 디버깅이 쉽다.
    - 단점
        - 실행 속도가 상대적으로 느리다.
